    Helper functions:
    - `def _numeric_params(family: SensorFamily) -> list[str]`: Returns keys from FAMILY_SCHEMAS[family] where the expected type is float or (int, float), filtering out str-typed params (Research Pitfall 4).
    - `def _build_search_bounds(family_ranges: SensorFamilyRanges, param_names: list[str]) -> np.ndarray`: Shape (D, 2). For each param, use ParameterRange.min and .max. If range spans >100x (max/min > 100), use log10 scale. If min or max is None, fall back to typical * 0.5 and typical * 2.0. Returns raw physical bounds (normalization to [0,1] done separately).
    - `def _normalize(values: np.ndarray, bounds: np.ndarray) -> np.ndarray`: Maps physical values to [0,1] using bounds. Log-scale params already converted.
    - `def _denormalize(normalized: np.ndarray, bounds: np.ndarray) -> np.ndarray`: Maps [0,1] back to physical values.
    - `def _build_sensor_at_point(base_sensor: SensorNode, param_names: list[str], physical_values: np.ndarray) -> SensorNode`: Creates a new SensorNode with updated family_specs using model_copy. Only updates numeric params; string params kept from base_sensor.
    - `def _evaluate_point(sensor: SensorNode, estimation_task: str, constraints: dict[str, float | str], cost_weights: CostWeights, family_cost_range: tuple[float, float], family_power_range: tuple[float, float], family_weight_range: tuple[float, float]) -> tuple[float, float, float]`: Returns (crb_bound, operational_cost, neg_constraint_margin). crb_bound from compute_crb(sensor, estimation_task).bound_value. operational_cost from compute_operational_cost. constraint_margin = min(s.margin for s in check_constraints(sensor, constraints)) if constraints else 0.0; negate for minimization. If CRB is inf, return (inf, inf, inf) to exclude point.

    Main functions:
    - `def _optimize_family(base_sensor: SensorNode, family_ranges: SensorFamilyRanges, estimation_task: str, constraints: dict[str, float | str], cost_weights: CostWeights) -> FamilyOptimizationResult`:
      1. Get numeric_params via _numeric_params. If empty, evaluate base_sensor once and return single-point result.
      2. Build search bounds. Compute family cost/power/weight ranges from base_sensor.operational.
      3. Generate initial samples via LatinHypercube(d=len(numeric_params)).random(n=max(10, 2*len(numeric_params))). Denormalize to physical space.
      4. Evaluate initial samples: for each, build sensor, evaluate point, record objectives.
      5. Use scalarized multi-objective BO: generate 10 scalarization weight vectors via _generate_scalarization_weights(10). For each weight vector:
         a. Scalarize objectives: weighted_obj = w1*crb_norm + w2*cost_norm + w3*neg_margin_norm (normalize each dim to [0,1] from observed range).
         b. Fit MinimalGP on normalized X -> scalarized_obj.
         c. BO loop: optimize_acquisition to find next point, evaluate, update GP, check should_stop.
      6. Collect all evaluated points. Filter inf CRB. Extract Pareto front via extract_pareto_front.
      7. Build ParetoPoint for each front member. Build BOMetadata.
      8. Return FamilyOptimizationResult.

    - `def optimize_sensors(feasibility_result: FeasibilityResult, scope: str = "medium", task: str = "", constraints: dict[str, float | str] | None = None, cost_weights: CostWeights | None = None) -> OptimizationResult`:
      1. If no ranked_configs, return empty OptimizationResult.
      2. Load family_ranges via load_family_ranges().
      3. Group ranked_configs by sensor_family. For each family, pick the top-ranked SensorConfig.
      4. For each family: lazy-load sensor from config, call _optimize_family.
      5. Collect FamilyOptimizationResult tuple. Build OptimizationResult with totals.
      6. Apply filter_by_scope(result, scope).
      7. Return final OptimizationResult.

    Note: The function needs SensorNode instances, but FeasibilityResult only has SensorConfig (which has sensor_name + family). The optimizer must reconstruct SensorNode from the sensor YAML loader. Use `from agentsim.knowledge_graph.loader import load_sensors_for_family` or construct from YAML. If that function does not exist, build a helper `_load_sensor_by_name(name, family)` that loads the family YAML and finds the matching profile.
