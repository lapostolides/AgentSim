    Create NEW file src/agentsim/physics/context.py with these functions:

    **1. `format_physics_context(domain, paradigm=None, sensor_catalog=None) -> str`:**
    The base formatter. Renders a complete physics context section:
    - Section header: `## {domain.domain} Physics Context` (uses domain.domain string, e.g., "nlos_transient_imaging")
    - Domain description: domain.description
    - Governing equations: iterate domain.governing_equations, format name, description, latex
    - Dimensionless groups: iterate domain.dimensionless_groups, format name, formula, description
    - Reconstruction algorithms: iterate domain.reconstruction_algorithms, format name, reference, spatial_resolution
    - If paradigm provided:
      - Paradigm section: `### {paradigm.paradigm} Paradigm` with description
      - Geometry constraints: iterate paradigm.geometry_constraints dict, format each key and its sub-dict values
      - Transfer functions: iterate paradigm.transfer_functions, format input->output with relationship and formula
      - Published baselines: iterate paradigm.published_baselines
    - If sensor_catalog provided:
      - Sensor section: `### Available Sensors`
      - For each compatible sensor (filter by paradigm.compatible_sensor_types if paradigm provided):
        format name, sensor_type, timing parameters, spatial parameters
    - Return joined lines as string. Return "" if domain is None.

    **2. `format_hypothesis_context(domain, paradigm=None) -> str`:**
    Wraps format_physics_context with hypothesis-specific framing:
    - Header: `## Physics Context for Hypothesis Generation`
    - Instruction: "Ground your hypothesis in the following physics. Use governing equations and dimensionless groups to constrain your proposed experiments."
    - Calls format_physics_context(domain, paradigm) for the content (no sensor_catalog — hypothesis agent doesn't need hardware details)
    - Return joined string

    **3. `format_analysis_context(domain, paradigm=None) -> str`:**
    Wraps format_physics_context with analysis-specific framing:
    - Header: `## Physics Context for Result Analysis`
    - Instruction: "When analyzing experiment results, check these physics expectations:"
    - Signal physics section (generic): "Signal should follow expected falloff patterns", "Peak locations should match expected path lengths"
    - If paradigm: "### Paradigm-Specific Validation" with geometry constraints as validation criteria
    - Reconstruction quality section from domain.reconstruction_algorithms (algorithm-specific checks)
    - Common failure modes section (generic)
    - Return joined string

    **4. `format_scene_context(domain, paradigm=None, sensor_catalog=None) -> str`:**
    Per D-06, the scene agent gets EVERYTHING:
    - Header: `## Physics Constraints for Scene Generation`
    - Instruction: "Your generated simulation code MUST respect these physics constraints. Violating them will cause validation failures."
    - Full physics context via format_physics_context(domain, paradigm, sensor_catalog)
    - If paradigm: explicit constraint summary listing each geometry_constraint with min/max/typical values as hard bounds
    - If paradigm and sensor_catalog: sensor selection guidance ("Compatible sensor types: {types}. Select a named sensor profile from the catalog.")
    - Published baselines section: "These published configurations are known-good starting points"
    - Return joined string

    All formatters use the SAME iteration logic over domain/paradigm/sensor models. No `if paradigm.paradigm == "relay_wall":` checks anywhere. The formatters are fully generic — they render whatever data is in the models.

    CRITICAL per D-07: No paradigm-specific Python formatting logic. The formatters iterate over dict keys, tuple entries, and model fields generically.
