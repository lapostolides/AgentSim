    **Changes to `_load_bundle_from_directory()`:**

    After loading sensor classes (step 3, lines 160-166), add inline profile extraction:

    ```python
    # 3. Sensor classes (top-level YAML in sensors/, excluding profiles/)
    sensor_classes: dict[str, SensorClass] = {}
    sensors_dir = domain_dir / "sensors"
    if sensors_dir.is_dir():
        for p in sorted(sensors_dir.glob("*.yaml")):
            raw = _load_yaml(p)
            sc = SensorClass.model_validate(raw)  # Pydantic ignores extra 'profiles' key
            sensor_classes[sc.name] = sc

            # NEW: Extract inline profiles from the same YAML
            for profile_data in raw.get("profiles", []):
                sp = SensorProfile.model_validate(profile_data)
                sensor_profiles[sp.name.lower().replace(" ", "")] = sp
    ```

    Move the `sensor_profiles` dict initialization BEFORE step 3 (currently it's initialized in step 4). Then in step 4 (profiles directory), only load profiles that aren't already loaded from inline data:

    ```python
    # 4. Sensor profiles — prefer inline, fall back to profiles/ directory
    profiles_dir = sensors_dir / "profiles" if sensors_dir.is_dir() else None
    if profiles_dir is not None and profiles_dir.is_dir():
        for p in sorted(profiles_dir.glob("*.yaml")):
            key = p.stem
            if key not in sensor_profiles:  # Don't overwrite inline
                sp = SensorProfile.model_validate(_load_yaml(p))
                sensor_profiles[key] = sp
    ```

    This approach:
    - SensorClass.model_validate() works because Pydantic ignores unknown fields (`profiles` key)
    - Inline profiles are extracted and validated as SensorProfile objects
    - profiles/ directory files serve as fallback (not overriding inline)
    - context.py and runner.py are untouched -- they receive the same SensorClass/SensorProfile/SensorCatalog types
