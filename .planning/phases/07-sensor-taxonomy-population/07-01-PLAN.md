    **Step 2: Create `src/agentsim/knowledge_graph/loader.py`**

    Two public functions:

    `load_sensors(families: tuple[SensorFamily, ...] | None = None) -> tuple[SensorNode, ...]`
    - Scans `_SENSORS_DIR = Path(__file__).parent / "sensors"` for `*.yaml` files
    - For each YAML: reads `family` field, converts to SensorFamily enum, skips if filtered out
    - For each entry in `sensors` list: builds SensorNode by mapping YAML sections to GeometricProps, TemporalProps, RadiometricProps, OperationalProps, and family_specs
    - CRITICAL: Coerce numeric family_specs values -- check FAMILY_SCHEMAS for the expected type. If schema expects `float` and YAML gave `int`, cast to `float`. If schema expects `(int, float)`, leave as-is.
    - CRITICAL: Coerce all numeric property group fields to `float` (YAML `safe_load` may return `int` for values like `64`)
    - Returns `tuple(results)` (immutable)

    `load_family_ranges(families: tuple[SensorFamily, ...] | None = None) -> dict[SensorFamily, SensorFamilyRanges]`
    - Scans same YAML directory
    - Parses `ranges` section into `ParameterRange` objects
    - Returns dict mapping family enum to SensorFamilyRanges

    Internal helper `_parse_sensor_node(data: dict, family: SensorFamily) -> SensorNode`:
    - Maps `data["geometric"]` -> GeometricProps (only include keys that GeometricProps accepts)
    - Maps `data["temporal"]` -> TemporalProps
    - Maps `data["radiometric"]` -> RadiometricProps
    - Maps `data.get("operational")` -> OperationalProps or None
    - Maps `data["family_specs"]` -> dict with type coercion
    - Sets `name=data["name"]`, `family=family`, `description=data.get("description", "")`

    Use `yaml.safe_load`, `structlog.get_logger()`, `from __future__ import annotations`.
