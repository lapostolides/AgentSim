Create `src/agentsim/knowledge_graph/units.py` with:
1. Module docstring: "Pint-based unit validation helpers for knowledge graph models."
2. `from __future__ import annotations` at top
3. `import pint` and `_ureg = pint.UnitRegistry()` (per D-04, same pattern as physics/models.py)
4. `CANONICAL_UNITS: dict[str, str]` mapping quantity categories to canonical unit strings:
   - "time": "picosecond", "angle": "degree", "length": "meter", "frequency": "hertz",
   - "ratio": "dimensionless", "power": "watt", "mass": "gram", "currency": "USD",
   - "temperature": "kelvin", "voltage": "volt"
5. `_EXPECTED_DIMENSIONALITY: dict[str, str]` mapping categories to Pint dimensionality strings:
   - "time": "[time]", "angle": "" (Pint treats angles as dimensionless),
   - "length": "[length]", "frequency": "1 / [time]", "ratio": "",
   - "power": "[length] ** 2 * [mass] / [time] ** 3", "mass": "[mass]",
   - "temperature": "[temperature]",
   - "voltage": "[length] ** 2 * [mass] / [current] / [time] ** 3"
6. `_KNOWN_ANGLE_UNITS: frozenset[str]` = frozenset({"degree", "radian", "arcminute", "arcsecond"})
   Used to handle the special case where Pint treats angles as dimensionless.
7. `def validate_unit(unit_str: str, quantity_category: str) -> None:` with Google-style docstring.
   - If quantity_category in ("currency", "ratio"): return immediately (skip Pint validation).
   - If quantity_category == "angle": check unit_str is in _KNOWN_ANGLE_UNITS or parseable as angle via Pint; raise ValueError if not.
   - Otherwise: parse unit_str via `_ureg.parse_units(unit_str)`, compare its `.dimensionality` string to `_EXPECTED_DIMENSIONALITY[quantity_category]`. Raise ValueError on mismatch. Catch `pint.UndefinedUnitError` and re-raise as ValueError with "Unknown unit" message.
